# WorkiFy — Система управління змінами з ШІ

## 🎯 Про проект

**WorkiFy** — веб-застосунок для управління робочими змінами персоналу з вбудованим ШІ-асистентом, командним чатом та базою даних працівників. Менеджери та працівники можуть переглядати розклад, подавати заявки, виявляти конфлікти та спілкуватися через вбудований чат. ШІ-агент дозволяє керувати системою через природну мову українською — без кліків по формах.

## 🗂️ База даних (employees.json)

Файл `data/employees.json` є центральним сховищем даних:

### Структура:

```json

```

## 🤖 ШІ-Асистент

### Два режими роботи:

#### 🧠 Локальний режим (без API ключа)

- Працює повністю офлайн
- Відповіді на основі `data/ai-responses.json`
- Розуміє запити про розклад, конфлікти, навантаження
- Може аналізувати дані по конкретних працівниках

#### ✨ Gemini AI (потребує API ключ)

- Повноцінний Function Calling агент
- Gemini 2.0 Flash з підтримкою інструментів
- Природна мова — виклики реальних функцій системи

### Доступні інструменти ШІ:

| Інструмент              | Опис                                 |
| ----------------------- | ------------------------------------ |
| `get_all_shifts`        | Всі зміни (з фільтром по працівнику) |
| `detect_conflicts`      | Виявлення накладань у розкладі       |
| `get_workload_stats`    | Аналіз годин навантаження            |
| `add_shift`             | Додавання зміни (тільки менеджер)    |
| `get_pending_requests`  | Активні заявки на розгляді           |
| `approve_shift_request` | Схвалення заявки (тільки менеджер)   |
| `reject_shift_request`  | Відхилення заявки (тільки менеджер)  |
| `check_shift_conflict`  | Перевірка конкретного часового слоту |
| `generate_request_text` | Генерація тексту заяви (3 стилі)     |

### Приклади запитів:

```
Покажи всі зміни цього тижня
Перевір конфлікти в розкладі
Скільки годин відпрацювала Анна?
Є активні заявки?
Напиши заяву на відпустку з 10 по 20 лютого через сімейні обставини
Додай зміну для Каріни на 2026-02-22 з 09:00 до 17:00  (тільки менеджер)
```

## 🚀 Запуск проекту

### Вимоги

- Сучасний браузер (Chrome, Firefox, Edge, Safari)
- Локальний сервер (через Live Server або Python)
- Для Gemini AI: Google AI Studio API ключ

### Локальний запуск

**VS Code Live Server (рекомендовано):**

```bash
# Встановіть розширення "Live Server" у VS Code
# Клікніть правою кнопкою на index.html → Open with Live Server
```

**Python HTTP Server:**

```bash
cd workify/
python3 -m http.server 3000
# Відкрийте http://localhost:3000
```

**Node.js npx serve:**

```bash
npx serve workify/
```

> ⚠️ Не відкривайте `index.html` напряму через `file://` — браузер заблокує запити до `data/*.json`

## 🔑 Налаштування Gemini AI (опціонально)

1. Перейдіть на [aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey)
2. Увійдіть через Google акаунт
3. Натисніть **Create API key** → **Create API key in new project**
4. Скопіюйте ключ (`AIzaSy...`)
5. У застосунку: ШІ-Асистент → **✨ Gemini AI** → введіть ключ → **Підключити**

> 💡 **Безкоштовний ліміт:** 15 запитів/хвилину, 1500 запитів/день

> 🔒 Ключ зберігається лише у `localStorage` браузера
